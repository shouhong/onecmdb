OneCMDB Models

Core

Transform_Model.xml

Includes object definitions for all transformations.
If any transformations, used in MDRs, should be performed
this model needs to be imported.

NMAP

NMAP_Model.xml

Includes definitions for NMAP model.

NMAP2BasicModel_MDR_Model.xml

Defines the MDR to transform NMAP model data into Basic Data model.
It will transform:
 NMAP_Host to Server CI.
 NMAP_Network to Network CI
 NMAP_[TCP/UDP]_Port to ServerApplication CI.

NMAP_MDR_Model.xml

Defines the MDR to execute NMAP and transform the result into the NMAP model.

Nagios

Nagios_MDR_Model.xml

Defines the MDR to import/export nagios config files.
Also defines the MDR to transform data between nagios and basic model.

Nagios_Model.xml

Defines the Nagios CI model. It's uses Version 3.x of nagios object definitions.

Nagios2BasicModel_MDR_Model.xml

Defines the MDR to transform Basic Model data to Nagios data and vice versa.

Basic

Basic_Model.xml

The Basic CMDB CI Model. It defines the core CI's like:
Application
OS
System
Component (hardware)
Company
Location
Maintenance
Service
etc.

Basic_Model_References.xml

The Basic CMDB Reference Model. It defines the refernce types between CI's like:
InstalledOn
DependensOn
PointsTo
BelongsTo
etc.

Demo

Import2BasicModel_MDR_Model.xml

Includes mdr definitions for import and transformation of external data.
Support for xml/csv and JDBC(SQL) data sources are available.

Demo_Basic_Model_Instances.xml

Demo data that populates the Basic Model with demo instances.
A good way to understand the basic model.

Demo_User_Role_Instances.xml

Defines three roles:
Admin
User
Guest
 And three users belonging to each role.
cmdb-admin (pasword=1qaz)
cmdb-user (pasword=1qaz)
cmdb-guest (pasword=1qaz)
